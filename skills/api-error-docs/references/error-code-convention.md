# Error Code Convention

## 코드 체계

에러 코드는 **4자리 숫자 문자열**로 관리한다.

### 범위 규칙

| 범위 | 용도 | 예시 |
|------|------|------|
| 3xxx | 도메인 비즈니스 예외 | 3001: 존재하지 않는 활동 상태 |
| 4000~4099 | 인증/인가 비즈니스 예외 | 4008: 계정 없음, 4009: 아이디 중복 |
| 4100~4199 | 회원가입 Validation | 4100: 아이디 필수, 4103: 비밀번호 규칙 |
| 4200~4299 | (다음 도메인 Validation) | 확장용 |

### 새 코드 부여 규칙

1. 기존 코드와 **절대 중복 금지** - 먼저 사용 중인 코드를 확인할 것
2. 같은 도메인/기능의 Validation은 **연속 번호**로 부여
3. 비즈니스 예외는 해당 도메인 범위 내에서 부여

## 메시지 형식 (`코드:메시지` 패턴)

Validation 어노테이션의 `message`에 `"코드:한글메시지"` 형식으로 작성한다.
`GlobalExceptionHandler`가 `:` 기준으로 파싱하여 코드와 메시지를 분리한다.

```java
// Good
@NotBlank(message = "4100:아이디는 필수 입력값입니다")

// Bad - 코드 없음
@NotBlank(message = "아이디는 필수 입력값입니다")
```

## 기존 등록 코드 (참고)

신규 코드 부여 전 반드시 프로젝트 내 기존 코드를 검색할 것:
```
grep -r "FAIL_CODE\|message = \"[0-9]" src/main/java/
```
